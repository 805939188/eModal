How to use

Simple way - Default title
var url = "/views/portview.html";
eModal.ajax(url);
Simple way - Custom title
var url = "/views/portview.html";
eModal.ajax(url, "XHR Title");
Tip: You can add custom buttons to footer!
Full custom modal
var options = {
        url: "/views/portview.html",
        title:'Header title',
        size: 'sm'
        subtitle: 'smaller text header',
        buttons: [
            {text: 'Ok', style: 'info',   close: true, click: eventA },
            {text: 'KO', style: 'danger', close: true, click: eventB }
        ],
        ...
    };
                 
eModal.ajax(options);

By default, the ajax modal does not define buttons! The loading animation is a progress-bar-striped from Bootstrap, but you can customize it by changing the eModal default settings to your custom HTML.

Alert


eModal.alert(), this creates a non-blocking alert using a Bootstrap modal!


Alert modal

An amazing message!

How to use

Simple way - Default title
var message = "Wello World!";
                         
eModal.alert(message);
Simple way - Custom title
var url = "Wello World!";
var title = "Alert Title";
                         
eModal.ajax(message, title);
Tip: You can use DOM elements as messages!
Message using DOM element
var options = {
        message: $("#my-element-id"),
        title: 'Header title',
        size: 'lg',
        subtitle: 'smaller text header',
        useBin: true
        ...
    };
                 
eModal.alert(options);
The element with ID my-element-id will be moved into the dialog's body, you can set the parameter useBin if you want to preserve it for reuse. If useBin is set to true, after close the dialog the element will be sent to a #recycle-bin container and all events will be preserved. If you set it to false, it will be deleted from the DOM so it's not possible to reuse it.

Confirm


eModal.confirm(), do you need user feedback? There is a nice way to do it!


Confirm modal

Is this dialog cool?

How to use

Simple way - Default title
var message = "Wello World!";
                         
eModal.confirm(message, null, callback);
Simple way - Custom title
var url = "Wello World!";
var title = "Alert Title";
                         
eModal.confirm(message, title, callback);
Tip: You can use change the default label name! { label: 'Yes' | 'True'| 'OK' }
Message using alternative lable
var options = {
        message: "The famouse question?",
        title: 'Header title',
        size: 'sm',
        callback: function(result) { result ? doActionTrue(result) : doActionFalse(); },
        subtitle: 'smaller text header',
        label: "True"   // use the possitive lable as key
        ...
    };
                 
eModal.confirm(options);
The callback is a mandatory argument. If it is not provided the dialog will still open, but an exception will be thrown when it is hidden. The callback will recieve a boolean argument, just like in the standard JavaScript confirm.

Prompt


eModal.prompt(), do you want to prompt for a response? Do it in a simple form!


Prompt modal

How to use

Simple way - Default title
var message = "Wello World!";
                         
eModal.prompt(message, null, callback);
Simple way - Custom title
var url = "Wello World!";
var title = "Alert Title";
                         
eModal.prompt(message, title, callback);
Tip: You can use the title to write the prompt's question!
Message using alternative lable
var options = {
        message: "The famous question?",
        title: 'Header title',
        size: 'sm'
        callback: function(result) { result ? doActionOk(result) : doActionCancel(); },
        subtitle: 'smaller text header'
        ...
    };
                 
eModal.prompt(options);
The callback is a mandatory argument. If it is not provided the dialog will still open, but an exception will be thrown when it is hidden. The callback will recieve a boolean argument, just like in the standard JavaScript confirm.

Options

Name Type Default Description
title string 'Attention' The header title string
message string, DOM object or jQuery object The body message string or the html element. eg: $(selector)
subtitle string The header subtitle string. This apper in a smaller text
size string Defines the size of modal [sm | lg]
loading bool false set loading progress as message
useBin bool false set message as recycable
css object CSs to apply to message body; only available if the message is a string
